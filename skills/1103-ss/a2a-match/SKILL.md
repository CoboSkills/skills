# A2A Match - 智能供需匹配平台

> **关键词**: 供需匹配 | 资源对接 | 能力匹配 | 商务合作 | 合伙人 | 需求发布 | 项目合作 | GPU算力 | 技术开发 | 电商货源 | 工厂OEM | 副业 | 找搭子 | 匹配引擎 | 智能撮合

---

## 🎯 一句话介绍

**零配置的智能匹配系统** —— 从对话中自动识别你的需求/能力/资源，帮你找到合适的合作伙伴。支持本地匹配（默认）和云端匹配（可选）。

## ⚠️ 隐私与数据说明（必读）

### 本地模式（默认）
- 所有数据存储在本地 `~/.qclaw/workspace/a2a/` 目录
- **不连接任何云端，不上传任何数据**
- 完全离线可用

### 云端模式（可选，需主动开启）
启用后才连接云端服务器 `http://81.70.250.9:3000`。

**云端同步的数据范围：**
| 数据 | 说明 |
|------|------|
| 你的 UserId | 自动生成（UUID，无个人信息） |
| 昵称 | 你填写的昵称 |
| 标签（Tags） | 从对话中提取的关键词，如 `GPU算力`、`Python开发` |
| 需求（Needs） | 你明确说"我需要..."的内容 |
| 资源（Resources） | 你明确说"我有..."的内容 |

**云端不上传的内容：**
- ❌ 你的 MEMORY.md 内容
- ❌ 你的每日记忆文件
- ❌ 任何包含个人隐私的信息（如邮箱、手机号除非你主动填写）

**云端认证：**
- 云端 API 支持 API Key 鉴权（生产模式）
- WebSocket 频道按 userId 隔离

**如何启用/关闭云端：**
编辑 `~/.qclaw/workspace/a2a/cloud_config.json`，将 `cloud.enabled` 改为 `false` 即可关闭。

- **能力匹配**: 店铺运营、直播带货、供应链管理

## 🔄 工作流程

```
用户对话 → 自动识别 → 存入档案 → 心跳检测 → 发现匹配 → 主动提示
```

1. **自动识别**: 从对话中识别需求/能力/资源
2. **存入档案**: 结构化存储到本地
3. **心跳检测**: 每30分钟检测匹配机会
4. **发现匹配**: 扫描其他用户的档案
5. **主动提示**: "我发现有人需要你的帮助！"

## 🧭 触发机制（核心！AI 必读）

### 触发条件

**不是每个对话都要处理，而是只在以下两种时机主动触发：**

---

#### 触发1️⃣：新用户首次对话

当检测到这是用户第一次使用 A2A Match（本地档案为空）时，主动发起引导：

```
🎯 欢迎使用 A2A Match！

这是一个智能供需匹配工具——能帮你找到需要你资源的人，也能帮你找到你需要的资源。

现在只需要告诉我一件事：
【你现在最想找什么？或者你有什么可以分享的？】

直接用自然语言说就行，比如：
• "我想找GPU算力"
• "我有技术团队，可以接开发项目"
• "我在找电商分销渠道"

说不清楚也没关系，边用边补充！
```

---

#### 触发2️⃣：对话中捕捉到信号

日常对话中，只要检测到以下关键词/句式，**立即触发识别流程**：

**需求信号**（用户想要什么）：
- "我想找……" / "我需要……" / "我在找……"
- "有没有人能……" / "谁有……" / "求推荐……"
- "想合作……" / "想对接……" / "想认识……"
- "需要能……的人/团队/资源"

**资源/能力信号**（用户能提供什么）：
- "我有……" / "我们可以……" / "我司……"
- "能提供……" / "可承接……" / "可支持……"
- "有xxx资源/渠道/人脉"
- "团队能……" / "擅长……"

---

#### 触发后：识别 → 确认 → 发布

检测到信号后，AI 按以下步骤操作：

**Step 1：识别提取**
用自然语言复述用户说的事实，不推断、不添加：
```
"我听到你说：你想找 GPU 算力来训练模型。是这样吗？"
```

**Step 2：确认发布**
用户确认后，更新本地档案并提示：
```
✅ 已记录你的需求：GPU算力（训练模型用）

已加入匹配候选。下次有心跳检测时会帮你扫描匹配。
如果想现在就看有没有合适的，直接说"帮我看看匹配"。
```

**Step 3：持续完善**
如果后续对话中用户补充了更多细节，也按同样流程处理，不遗漏、不重复。

---

### 防幻觉原则

| 用户说的话 | ✅ 记录 | ❌ 不推断 |
|-----------|--------|----------|
| "我需要GPU算力" | 需求: GPU算力 | - |
| "我在做AI项目" | - | ❌ 不推断需要GPU |
| "我是产品经理" | 职业: 产品经理 | ❌ 不推断懂用户研究 |
| "我有5年Python经验" | 能力: Python (5年) | - |

**置信度阈值**: 只记录置信度 > 80% 的信号

## ☁️ 云端服务（可选）

| 项目 | 值 |
|------|----|
| **API 地址** | `http://81.70.250.9:3000` |
| **WebSocket** | `ws://81.70.250.9:3000` |
| **匹配阈值** | 0.3（30% 以上即推送） |
| **数据库** | MongoDB |
| **鉴权** | API Key（生产模式）/ 开放（开发模式） |

### 云端开关

默认关闭！编辑 `~/.qclaw/workspace/a2a/cloud_config.json`：

```json
{
  "cloud": {
    "enabled": false   // 改为 true 开启云端
  }
}
```

### REST API

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/profile` | 创建/更新档案并触发匹配 |
| GET | `/api/profile/:userId` | 获取指定用户档案 |
| GET | `/api/matches/:userId` | 获取用户的所有匹配 |
| POST | `/api/match/:id/accept` | 接受匹配 |
| POST | `/api/match/:id/reject` | 拒绝匹配 |

### WebSocket 连接示例

```javascript
const io = require('socket.io-client');
const client = io('http://81.70.250.9:3000');

client.on('connect', () => {
    client.emit('join', '你的userId');  // 加入通知频道
});

client.on('new_matches', (data) => {
    console.log('🎉 新匹配通知:', data);
});

client.on('match_accepted', (data) => {
    console.log('✅ 对方接受了匹配!');
});
```
