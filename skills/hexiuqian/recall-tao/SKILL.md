# 抖音自动回复技能

通过浏览器自动化实现抖音评论的智能自动回复。

## 核心能力

1. **连接抖音创作者中心** - 自动打开并登录 creator.douyin.com
2. **多视频监控** - 同时监控多个视频/帖子的评论
3. **智能分析** - 理解用户评论内容和意图
4. **自动回复** - 生成积极、大方、热情、合法的回复

## 快速开始

### 步骤1：启动浏览器并登录

```
1. 用户请求"帮我连接抖音账号"
2. 调用 browser 工具启动 openclaw 独立浏览器
3. 导航到 https://creator.douyin.com
4. 用户扫码登录
5. 确认登录成功后进入互动管理页面
```

### 步骤2：开始监控

进入评论管理页面，开始监控新评论。

## 浏览器启动配置

使用 openclaw 独立浏览器配置：

```
browser action=start profile=openclaw
```

关键参数：
- `profile`: 使用 "openclaw" 独立浏览器
- 不使用 "user" 或 "chrome-relay"，避免与用户日常浏览器冲突

## 页面导航路径

登录后导航路径：

```
首页 → 互动管理 → 评论管理
```

评论管理URL：`https://creator.douyin.com/creator-micro/interactive/comment`

## 多视频监控

抖音创作者中心可以同时发布多个视频/帖子，监控时需要处理所有视频的评论。

### 评论管理页面结构

```
评论管理页面
├── 视频筛选区（顶部）
│   ├── 全部视频（默认）
│   └── 单个视频筛选
│
└── 评论列表
    ├── 视频1的评论
    ├── 视频2的评论
    └── 视频3的评论
    ...
```

### 多视频监控策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 全量监控 | 监控所有视频的评论 | 视频数量少，评论量适中 |
| 分页轮询 | 分批次检查各视频 | 视频数量多，评论量大 |
| 热点优先 | 优先监控热门视频 | 部分视频评论量大 |

## 回复原则

对所有用户评论，智能分析后生成回复，遵循以下原则：

| 原则 | 说明 |
|------|------|
| 积极 | 正面回应，传递正能量 |
| 大方 | 不卑不亢，得体有度 |
| 热情 | 亲切友好，拉近距离 |
| 合法 | 遵守法律法规，符合平台规范 |

## 评论回复流程

### ⚠️ 关键：正确的回复方式

抖音创作者中心的评论回复**必须**使用嵌套输入框，不能使用页面顶部的通用输入框。

**正确流程：**

1. **点击目标评论的"回复"按钮**（不是顶部的输入框）
2. 等待嵌套输入框展开（placeholder 显示 "回复 @用户名："）
3. 在嵌套输入框中输入回复内容
4. 点击该输入框旁边的"发送"按钮

**错误做法：**
- ❌ 在页面顶部的输入框（placeholder "有爱评论，说点好听的~"）输入 → 会发成独立新评论
- ❌ 不点击"回复"按钮直接输入 → 内容发送到错误位置

### DOM 结构识别

```
评论列表
├── 用户评论
│   ├── 用户名 + 时间 + 内容
│   └── 操作区
│       ├── 点赞
│       ├── "回复"按钮 ← 必须点击这个
│       ├── 删除
│       └── 举报
│
└── 展开后的回复区（点击"回复"后出现）
    ├── 输入框（contenteditable，placeholder="回复 @用户名："） ← 在这里输入
    ├── 取消按钮
    └── 发送按钮 ← 点击这个发送
```

### JavaScript 输入方法

抖音创作者中心使用 contenteditable div 作为输入框，直接设置 innerText 可能无法触发 React/Vue 的状态更新。

**正确方法：使用 execCommand**

```javascript
// 步骤1: 点击评论的"回复"按钮
// 通过 browser act click ref: "回复按钮的ref"

// 步骤2: 找到嵌套的回复输入框（第二个 contenteditable）
const editables = document.querySelectorAll('[contenteditable="true"]');
const replyInput = editables[1]; // 第二个是回复输入框
// editables[0] 是页面顶部的通用评论输入框，不能用！

// 步骤3: 输入内容
replyInput.focus();
document.execCommand('insertText', false, '回复内容');
replyInput.dispatchEvent(new Event('input', { bubbles: true }));

// 步骤4: 点击发送按钮
// 通过 browser act click ref: "发送按钮的ref"
```

## 身份验证处理

发送回复时可能触发身份验证弹窗：

```
验证方式：
1. 接收短信验证
2. 扫码验证
```

检测到验证弹窗时，提示用户完成验证后继续。

## 监控循环

使用定时任务定期检查新评论：

```
1. 每 N 秒刷新页面（建议 30-60 秒）
2. 解析页面获取新评论
3. 对每条新评论进行智能分析
4. 生成符合原则的回复内容
5. 执行回复操作（正确使用嵌套输入框）
6. 记录已处理的消息ID，避免重复
```

## ⚠️ 平台限制

1. **每条评论只能回复一次** - 抖音平台限制每条评论只能有一条作者回复，无法多次回复
2. **回复要"一次到位"** - 由于只能回复一次，内容要完整有效

## 注意事项

1. **频率控制** - 避免过于频繁的刷新和回复，可能触发风控
2. **回复延迟** - 建议在收到消息后等待几秒再回复，模拟真人行为
3. **内容审核** - 回复内容需符合平台规范，避免敏感词
4. **登录状态** - 长时间运行可能需要重新登录
5. **一次回复原则** - 每条评论只有一次回复机会，内容要完整有效
6. **必须使用嵌套输入框** - 点击"回复"按钮后展开的输入框才是正确的

## 错误处理

| 错误 | 处理方式 |
|------|----------|
| 浏览器连接失败 | 检查浏览器进程，重启浏览器 |
| 登录过期 | 提示用户重新扫码登录 |
| 身份验证弹窗 | 等待用户完成验证 |
| 回复发送失败 | 检查网络，重试或提示用户 |
| 元素找不到 | 刷新页面后重试 |
| 回复发错位置 | 删除错误评论，使用正确流程重发 |

## 监控工作流

### 完整监控循环

```
1. 导航到评论管理页面
   browser navigate https://creator.douyin.com/creator-micro/interactive/comment

2. 获取页面快照
   browser snapshot

3. 解析评论列表
   - 识别新评论（未回复的，即有"回复"按钮的）
   - 记录评论ID避免重复处理

4. 对每条新评论：
   a. 智能分析评论内容
   b. 生成积极、大方、热情、合法的回复
   c. 点击评论的"回复"按钮（重要！）
   d. 等待嵌套输入框展开
   e. 在嵌套输入框中输入内容
   f. 点击发送按钮
   g. 处理可能的身份验证
   h. 确认发送成功

5. 等待一段时间后重复步骤1
```

## 常见问题

### Q: 回复显示在评论列表顶部而不是评论下方？

A: 这是因为使用了页面顶部的通用输入框。必须点击目标评论的"回复"按钮，在展开的嵌套输入框中输入。

### Q: 如何删除错误发送的评论？

A: 点击错误评论下的"删除"按钮，确认删除后重新用正确流程回复。

### Q: 评论列表中回复和评论如何区分？

A: 正确的回复会嵌套显示在用户评论下方，有"收起"按钮可以折叠。独立评论会显示在评论列表的顶部位置。
