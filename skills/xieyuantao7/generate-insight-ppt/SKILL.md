# PPT Insight Generator Skill

## 概述

自动生成高质量技术洞察演示文稿，采用**深红色主题色 + 白色背景**的专业设计。每页幻灯片都包含：
- 🎯 **核心观点** - 明确的洞察结论
- 📊 **数据可视化** - 图表、统计数字
- ⚖️ **对比分析** - 前后对比、竞品对比、趋势对比
- 💡 **深度洞察** - 超越表面的技术分析

## 使用方法

### 基本用法

```bash
node generate_insight_ppt.js --topic "主题名称" --output "文件名.pptx"
```

### 参数说明

| 参数 | 必需 | 说明 |
|------|------|------|
| `--topic` | ✅ | PPT 主题名称 |
| `--output` | ❌ | 输出文件名（默认：insight_presentation.pptx） |
| `--slides` | ❌ | 幻灯片数量（默认：10） |
| `--content` | ❌ | JSON 格式的内容大纲 |

## 幻灯片模板类型

### 标准页面结构逻辑

**每页遵循统一的逻辑框架：**

```
┌─────────────────────────────────────┐
│  【标题】体现核心观点 (深红色 24pt)   │
├─────────────────────────────────────┤
│  【上部】一句话描述问题/挑战/目标    │
│  (淡灰色文本框，可加透明图装饰)      │
├─────────────────────────────────────┤
│  【中部】技术架构/数据对比/解决方案  │
│  (根据内容选择左右或上下排版)        │
│  - 左右排版：对比、并列关系          │
│  - 上下排版：流程、层级关系          │
├─────────────────────────────────────┤
│  【下部】一句话总结效果/目标达成     │
│  (淡灰色文本框，强调色点缀)          │
└─────────────────────────────────────┘
```

### 1. 封面页 (Title Slide)
- 大标题 + 副标题 (微软雅黑，深红色)
- 白色背景或淡灰色渐变
- 可添加透明装饰图
- 日期/作者信息

### 2. 目录页 (Agenda)
- 清晰的章节结构
- 编号 + 标题 + 简短描述
- 淡灰色文本框区分章节

### 3. 核心观点页 (Key Insight) - 标准结构
- **标题**: 核心观点陈述 (深红色 24pt)
- **上部**: 一句话描述问题/挑战 (淡灰文本框)
- **中部**: 支持观点的数据/事实 (左右或上下排版)
- **下部**: 一句话总结洞察 (淡灰文本框)

### 4. 数据对比页 (Data Comparison) - 标准结构
- **标题**: 对比结论 (如"A 方案性能提升 40%")
- **上部**: 对比场景说明 (淡灰文本框)
- **中部**: 柱状图/条形图 + 关键数字 (左右排版)
- **下部**: 对比结果的业务意义 (淡灰文本框)

### 5. 趋势分析页 (Trend Analysis) - 标准结构
- **标题**: 趋势判断 (如"用户量持续增长")
- **上部**: 分析时间范围 (淡灰文本框)
- **中部**: 折线图 + 增长率标注 (上下排版)
- **下部**: 趋势预测或建议 (淡灰文本框)

### 6. 占比分析页 (Breakdown) - 标准结构
- **标题**: 占比结论 (如"移动端占比超 60%")
- **上部**: 分析对象说明 (淡灰文本框)
- **中部**: 环形图/饼图 + 百分比 (左右排版)
- **下部**: 占比变化的意义 (淡灰文本框)

### 7. 技术架构页 (Architecture) - 标准结构
- **标题**: 架构核心价值 (如"微服务提升可扩展性")
- **上部**: 架构设计目标 (淡灰文本框)
- **中部**: 架构图 + 关键组件说明 (上下排版)
- **下部**: 架构带来的效果 (淡灰文本框)

### 8. 案例研究页 (Case Study) - 标准结构
- **标题**: 案例核心成果
- **上部**: 案例背景一句话 (淡灰文本框)
- **中部**: 关键数据 + 解决方案 (左右排版)
- **下部**: 可复用的经验 (淡灰文本框)

### 9. 总结页 (Summary)
- 核心要点回顾 (3-5 条)
- 行动建议
- 关键 takeaway

### 10. 结束页 (Closing)
- Q&A 提示
- 联系方式
- 参考资料

## 设计原则

### ✅ DO - 应该做的

1. **每页一个核心观点** - 避免信息过载
2. **数据驱动** - 用具体数字支撑观点
3. **对比清晰** - 使用颜色、大小区分对比项
4. **留白充足** - 0.5" 最小边距，0.3" 元素间距
5. **一致性** - 字体、颜色、图标风格统一
6. **视觉层次** - 标题 > 副标题 > 正文 > 标注
7. **标准页面逻辑** - 标题→上部说明→中部展开→下部总结
8. **字体规范** - 默认微软雅黑，标题深红色 24pt，正文 11-14pt
9. **文本框样式** - 淡灰色背景 (#F0F0F0)，可加透明度 80-90%
10. **排版决策** - 对比/并列用左右排版，流程/层级用上下排版

### ❌ DON'T - 避免的

1. **文字堆砌** - 每页不超过 6 行正文
2. **颜色杂乱** - 只用主色调 + 辅助色 + 强调色
3. **图表复杂** - 每个图表只传达一个信息
4. **字体混用** - 只用微软雅黑
5. **低对比度** - 确保文字清晰可读
6. **无来源数据** - 标注数据来源
7. **标题模糊** - 标题必须体现明确观点
8. **缺少上部说明** - 每页需有一句话问题/挑战/目标描述
9. **无下部总结** - 每页需有一句话效果/目标达成总结

## 故障排除

### 问题：生成的 PPT 无法打开
- 检查 pptxgenjs 是否正确安装
- 确保 Node.js 版本 >= 16

### 问题：颜色显示不正确
- 确认使用 6 位十六进制颜色代码
- 检查 PowerPoint 版本兼容性

### 问题：中文字体显示异常
- 确保系统安装了 Arial 和 Calibri 字体
- Windows 系统通常预装这些字体

## 最佳实践

1. **先规划内容结构** - 使用 JSON 大纲定义所有内容
2. **数据优先** - 收集并验证所有数据
3. **观点明确** - 每页只讲一件事
4. **视觉一致** - 使用统一的图标和颜色
5. **迭代优化** - 生成后审查并调整
