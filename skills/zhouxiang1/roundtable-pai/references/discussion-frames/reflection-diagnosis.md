# Reflection Diagnosis

适用：我为什么总这样、我到底卡在哪、根因是什么。

## 推荐轮数

- `5` 到 `7` 轮

## 节点结构

- `5` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 结案陈词
- `6` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 反转 / 修正 -> 结案陈词
- `7` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区（一） -> 深水区（二） -> 反转 / 修正 -> 结案陈词

用户可见标题只写 `第一轮` 到 `第七轮`，不要把这些节点名写给用户看。

## 开场引子

```text
你现在要先生成一段“反思诊断型”讨论的短开场。

任务：
1. 用 1 到 3 句把三位人物请到桌前。
2. 最好由 initiator 开口，也可以是极短旁白。
3. 点出这场讨论真正要诊断的，不是表面困惑，而是更深的卡点。
4. 开场要温和但不空，不要像系统在念提示词。
```

## 开题

```text
你现在要生成一场“反思诊断型”讨论的开题轮。

任务：
1. 不复述原问题，要指出更深一层的困住点。
2. 明确这场讨论要解决的是“真正卡点是什么”，而不是立刻给安慰或建议。
3. 开题保持短，但要把问题靶心钉住。
```

## 立场亮相

```text
你现在要生成”反思诊断型”的立场亮相轮。

任务：
1. 三位角色都必须先重定义问题，而不是立刻给建议。
2. 每位角色只能抓一个最关键的根因方向。
3. 三种重定义路径必须明显不同。
```

## 深水区

```text
你现在要生成”反思诊断型”的深水区轮。

任务：
1. 讨论要进入根因的代价、边界、长期后果与自我欺骗机制。
2. 三位角色必须把问题推进到更难面对的一层，而不是停留在表层行为解释。
3. 本轮结束后，用户应能看见”我为什么一直重复这个模式”的更深解释。
```

```text
任务：
1. 继续下潜到更深的动机、身份叙事、关系结构或环境反馈机制。
2. 不要重复上一轮，要让核心矛盾发生迁移或被重新命名。
3. 这轮通常用于高复杂度问题。
```

## 反转 / 修正

```text
你现在要生成”反思诊断型”的反转 / 修正轮。

任务：
1. 至少一位角色要修正自己对根因的最初判断。
2. 或承认原来以为的根因只是表象，把问题重新定义。
3. 让诊断真正从”聪明解释”推进到”更接近真实”。
```

## 结案陈词

```text
你现在要生成“反思诊断型”的结案陈词轮。

任务：
1. 每位角色只保留最后想让用户记住的一句诊断。
2. 不再开新根因。
3. 给系统总结和下一步动作留出自然收口。
```

## 散场过桥

```text
你现在要生成“反思诊断型”讨论在结论字段前的短散场。

任务：
1. 用 1 到 2 句把整场诊断先收住。
2. 默认由 initiator 或 synthesizer 开口，谁更自然就用谁。
3. 散场要有“话先落到这里”的停顿感，但不要在这里提前做完整诊断结论。
```
